# 故障排查

## 双击 N_m3u8DL-RE.exe 没反应

`N_m3u8DL-RE.exe` 是命令行程序，没有图形界面。没有输入参数时窗口会立即退出。请启动本 GUI，或在 PowerShell 中传入播放列表地址。

## 工具路径标红

推荐把下载器放在 GUI 同目录、`tools` 子目录，或任一固定磁盘根部的 `Downloads` / `Download` / `下载` 目录（例如 `E:\Downloads`）。FFmpeg 可以安装到系统 `PATH` 或 WinGet 默认位置：

```text
<程序目录>\N_m3u8DL-RE.exe
<程序目录>\tools\N_m3u8DL-RE.exe
%LOCALAPPDATA%\Microsoft\WinGet\Links\ffmpeg.exe
```

点击“自动检测”后可选择自动下载或浏览本机文件。GUI 自动下载会直连 GitHub Release，不使用 Windows 系统代理；失败时先确认当前网络可以直接访问 GitHub，再重试或改用“浏览...”。工具会保存到 `%LOCALAPPDATA%\N_m3u8DL-RE-GUI\tools`，不需要管理员权限。Release ZIP 用户也可以运行备用的 `Setup-dependencies.cmd`；该备用脚本遵循 PowerShell 和 WinGet 自身的网络设置。

## Blob 地址无法下载

`blob:` 是浏览器内存对象，不是网络 URL，因此粘贴到主输入框仍会被拒绝。请点击“从网页捕获”，在内嵌浏览器中打开视频页并播放；捕获到 Blob 后展开确认切片，选择“导入此 Blob 并下载”。详细步骤见[网页捕获与 Blob 播放列表](USAGE.zh-CN.md#3-网页捕获与-blob-播放列表)。

如果列表中只有普通请求、没有 Blob，先确认视频确实开始播放。Blob 必须由捕获窗口中的页面创建；外部 Chrome 或扩展进程已有的 Blob 无法跨进程读取，此时仍需复制扩展“原始m3u8”的完整 `#EXTM3U` 正文并点击“粘贴”。

## 捕获列表的总大小显示 403

捕获窗口读取播放列表正文发生在内嵌浏览器内部，但精确的“总大小”需要桌面程序另行请求播放列表和部分切片。CDN 如果校验浏览器会话、短期签名或 TLS 指纹，独立探测可能返回 403。只要“内容”列仍显示切片数，播放列表正文就已经捕获成功，不代表正文丢失。

如果捕获窗口同时取得了主播放列表和媒体播放列表，程序会使用主列表的 `AVERAGE-BANDWIDTH`（没有时使用 `BANDWIDTH`）乘以媒体列表全部 `#EXTINF` 时长，再除以 8，显示“约 XX MB/GB”。这个值不需要重新访问切片，因此不受上述 403 影响，但它是码率估算值，不是精确文件长度。没有可关联码率时才显示“未知（已捕获）”。

程序会把捕获到的请求头带给大小探测及下载阶段，但 Cookie、Referer、Origin、Authorization 和站点自定义头只会发送给与捕获 `SourceUrl` 同源的目标；来源不明、异域子列表、KEY 或重定向只保留 User-Agent。这个限制可能使跨域精确大小探测回退为码率估算，但不会影响已关联到真实同源分片的受保护下载。大小探测仍使用桌面 HTTP 客户端，因此 Cloudflare 按 TLS 指纹拒绝时也可能显示估算值；这不妨碍后续启用 cURL 通道。不要在日志、截图或 Issue 中公开请求头值。

## 下载器反复加载 playlist.m3u8 并返回 403

这表示 403 发生在真正下载阶段，而不是大小探测阶段。新版捕获窗口不会再把已经取得正文的普通 HTTP m3u8 URL直接交给下载器：媒体播放列表正文会补全地址并保存成本地文件；选中主播放列表时，会自动选择浏览器已经捕获到、实际包含切片的最高码率子播放列表。

使用正文模式后，下载日志中的第一条“加载URL”应指向 `%TEMP%\N_m3u8DL-RE-GUI\ImportedPlaylists\pasted_*.m3u8`，而不是原来的远程 `playlist.m3u8`。如果仍显示远程地址，说明正在运行旧版程序、播放列表正文尚未完成捕获，或选中主列表时子列表还没有出现；继续播放几秒后重新选择。

如果本地 m3u8 已成功加载，但随后具体的 `.ts` / `.m4s` / 伪装 `.jpeg` 地址返回 403，则 403 位于分片层。新版会在捕获窗口观察真实分片 Referer/UA；对 Surrit 或带 `#EXT-X-TOKEN` 的列表自动建立随机 `127.0.0.1` cURL 回环通道，日志会显示“已启用 Windows cURL 回环传输”。`N_m3u8DL-RE` 此时只访问本机并继续负责缓存和合并，Cloudflare 看到的是已经验证可用的 Windows cURL 客户端。

如果没有出现该日志，或程序提示缺少 Referer/User-Agent，请不要直接重试旧的本地 m3u8；重新点击“从网页捕获”，在内嵌浏览器中播放几秒，等提示“已捕获真实分片请求”后再选择。Windows 系统目录必须存在 `curl.exe`（当前受支持的 Windows 10/11 默认自带）。链接已经过期、账号会话无权访问或浏览器本身也无法播放时，本地通道不会绕过服务器权限。

下载过程中偶尔出现 cURL `exit 28`、连接超时、部分字节后停住或“主机中的软件中止了连接”，表示某一次 CDN 连接卡住，并不表示切片顺序错误。只要分片进度仍增加，下载器会保留已完成分片并补下失败项。1.3.3 起采用偏宽松的慢速策略：单次上游最多等待 300 秒，只有连续 60 秒低于 `512 B/s` 才认定连接实际卡死，并对连接、部分传输和接收类瞬态错误在代理内重试一次；受保护模式固定 4 个下载线程、每片最多 10 次下载器重试，本地请求等待 660 秒。1.4.0 会把已自动恢复的超时、重试和本地断开合并为周期计数，不再为每次尝试连续刷明显异常；只有请求交回下载器继续补片或整个任务最终失败时才保留重点信息。

## 播放列表包含相对地址

HLS 导入文本必须包含完整的 `https://` 分片、密钥、初始化文件和附加音轨地址。程序会拒绝：

```text
segments/001.ts
/video/001.ts
//cdn.example.com/video/001.ts
URI="key.key"
URI="init.mp4"
```

网页捕获模式会在页面提供 HTTP(S) Base URL 时自动补全这些地址。如果仍出现此提示，说明 Blob 来自没有可靠网络 Base URL 的页面（常见于扩展页），请复制已经补全 URL 的“原始m3u8”，或寻找真实的在线播放列表地址。

MPD 可以在作用域内提供绝对 `<BaseURL>https://cdn.example.com/video/</BaseURL>`，然后使用相对的 `media` / `initialization` 模板。没有绝对 BaseURL 的相对 MPD 引用仍会被拒绝。

## 日志提示“获取 KEY 失败”

单条 `ERROR` 不一定表示整个任务失败。先看最终退出状态和成品：

- 成品能完整播放、声音正常、可读到完整时长：通常不需要手动 KEY。
- 只有开头能播放、FFmpeg 报解密/数据错误：需要检查 KEY。
- 标准 AES-128 密钥 URL 可访问时，下载器通常会自动获取。

自动获取失败且你合法持有密钥时，点击“密钥...”，输入 32 位 HEX、16 字节 Base64 或选择密钥文件，然后从原播放列表重新处理。

## 下载显示 Done，但 TS 时长很短

TS 容器的时长探测可能只看到第一个时间戳周期。先使用“转换文件...”无损重新封装为 MP4。如果转换成功且输出时长完整，原始媒体内容是有效的，不需要密钥。

## 401、403 或连接失败

常见原因：

- 签名 URL 或 token 已过期。
- 服务器要求登录 Cookie、Referer 或特定 User-Agent。
- 密钥 URL 与媒体 URL使用不同权限。
- 代理、DNS 或防火墙阻止请求。

重新从浏览器获取最新播放列表地址。网页捕获会自动重放实际分片中的安全请求头，但不会猜测未在浏览器中出现的认证信息。为避免把 Cookie/Authorization 作为全局头发送给播放列表中的异源资源，直接输入远程 URL 时不会转发这类敏感凭据；需要登录态时请使用“从网页捕获”导入完整正文，由 cURL 回环通道按资源同源性发送。不要在公开 Issue 中粘贴 Cookie、Authorization、自定义 Token 或完整签名 URL；浏览器本身无权访问的资源也不会由 GUI 绕过。

## 退出代码为 0，但没有识别到输出

程序只把本次任务名对应且确实发生变化的媒体文件视为结果，避免误报保存目录中其他程序生成的文件。1.3.0 会保留该任务分片缓存；先点击“打开目录”检查是否存在未识别扩展名，也可以点击“重试下载”让下载器重新校验、补片和合并。

## 分片数量校验失败或网络中断

1.3.0 起，暂停、非零退出和“未识别到输出”都不会立即删除任务缓存。日志会显示 `%TEMP%\N_m3u8DL-RE-GUI\Downloads\download_<任务ID>` 形式的实际路径。网络恢复后点击“重试下载”，程序会使用相同 URL、保存名称和 `--tmp-dir`，完整分片会跳过，不完整或缺失分片会重新下载。

1.4.0 起，关闭应用或重启 Windows 后任务仍可恢复。程序启动时只恢复为暂停状态，不会自动联网；点击“继续”才复用原 `--tmp-dir`。恢复所需的 URL、播放列表、Cookie/Authorization、自定义头和 KEY/IV 由当前用户 DPAPI 加密，明文播放列表与临时密钥会在正常关闭时删除。有效未完成任务最多保留 3 天；点击“清除缓存”、确认成功或明确取消会立即清理。编辑输入链接只会标记任务变化；点击开始并确认切换到不同任务后，才删除旧缓存。修改保存目录、文件名、混流选项或手动密钥后再次下载，也会先放弃不匹配的旧缓存；单纯更换下载器或 FFmpeg 路径不会浪费已下载分片。

## Windows SmartScreen 提示未知发布者

本地构建 EXE 没有代码签名证书。确认文件来自你自己的构建或可信仓库后再运行。可以用 SHA-256 校验下载文件，并与 Release 附带的 `SHA256SUMS.txt` 对比：

```powershell
Get-FileHash .\M3U8视频下载器.exe -Algorithm SHA256
```

## 取消后仍有临时文件

从 1.2.2 起，每个下载任务都使用带所有权标记的独占临时目录。取消会通过 Windows Job Object 终止下载器和 FFmpeg 子进程，确认退出后只删除这个受管目录中的 `.ts`、`.m4s`、`.tmp` 等下载分片，同时清理 GUI 创建的播放列表和密钥临时文件。1.4.0 如果遇到文件暂时被占用，会保留所有权标记并写入待删除标记，下次启动继续清理，而不是把残留目录永久遗忘。

如果取消恰好发生在最终合并或直接录制阶段，上游可能已经在保存目录写入部分媒体文件。GUI 不会根据文件名猜测并删除保存目录内容，以免误删用户原有文件；这种情况应人工确认后处理。

## 下载器中文日志显示乱码

1.2.2 已按照当前 Windows 系统 ANSI 代码页读取 `N_m3u8DL-RE` 输出，同时继续按 UTF-8 读取 FFmpeg。旧版本出现 `����` 时请升级；`--no-ansi-color` 只控制颜色，不能修复编码。

## 下载进度在结束时一次性出现

部分 `N_m3u8DL-RE` 版本在输出被重定向时会删除进度重绘使用的换行和光标控制符。1.2.3 改为直接读取原始字符块，实时提取当前视频、音频或字幕轨的分片数和百分比；重复重绘不会再形成几十万字符的单行日志。真正开始二进制合并或混流后，进度条会切回活动状态，最终以“下载完成”或错误状态为准。
